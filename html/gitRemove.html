GitHub Remove Commands

Welcome to the fourth section where you will learn how to remove GitHub repositories. The reason you might want to delete a repository might be for many reasons, but knowing how is crucial. The first thing you will want to do before deleting a repository is removing git from your project. You can do that by typing the following command in your terminal(this action is permanent so be careful.):
rm -rf .git/
Once you have removed git from your project, you can move forward with removing your GitHub repository.

Go to your project on github.com and click on the settings section, and scroll all the way down until you see the danger section and click delete this repository.

You will be greeted with a modal to make sure you want to delete the repository and will require you to type in your username/project_name, and then hit the "I understand the consequences, delete the repository" button.
Once you confirm the deletion of your repository, its gone and you're done, you can create a new one and then reinitialize git on your local project.
