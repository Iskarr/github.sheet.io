GitHub Branching and Merging

1

Welcome to the third section where you will learn how to manage GitHub branches and merging those branches into your main branch. What is a branch?
Think of a tree, you have a lot of branches and so can your repositories. When you initialized your first GitHub project you created a branch called "main," that should be your current branch.
lets first learn how to check which branch you are working in. Type the following command inside the terminal you had open from the previous module:
git branch
You should see the name of the current branch you are working in with an asterick next to it.

2

Now that you know how to see which branch you are working it, its time to create a new one and move over to that new branch. Use the following commands to create a new branch called block2 and move to that new branch.
git branch block2
git checkout block2
(running this again command will verify that you are on block2 with an asterick next to it.)
git branch

So what happens if you made a mistake in naming your branch? You have two options. Your first option is to rename the branch or delete the branch and start over.
make sure to add a space between your old branch name and the name you want to replace the old name with.
git branch -m old_branch_name new_branch_name
If you prefer to just start over, you can just delete a branch by using the following command:
git branch -d branch_name

something that you can run is git checkout -b BRANCH_NAME (use -b with git checkout to both create a new branch and check it out at the same time.)

Now that you can create, checkout, rename, and delete branches, what is next? The next step would be to merge and push those new branches up to your GitHub project. So how do you go about merging a branch and why would you?
The reason is quite simple, you might have different versions of your application saved and in case something goes horribly wrong, you want a branch you can go back too. For example, if you wanted to update the main application branch you would checkout main and then run this command:
git merge branch_name
So when you merge, remember that you want to checkout the branch you plan to merge changes to NOT from. At this point you can just push up to your repository just use the branch you're on or checkout another branch to push as well.

Remember that you cannot delete a branch that you are currently working in, check where the arrow in the image is.
